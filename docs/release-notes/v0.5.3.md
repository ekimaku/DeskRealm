# DeskRealm v0.5.3 — Display topology aware icon layouts

This release hardens Desktop icon layout persistence for multi-monitor, resolution-change and scaling-change scenarios.

## Changed

- Icon layouts now store display topology variants per virtual desktop.
- The topology key includes active screens, virtual bounds, screen resolution, orientation and effective DPI / scale.
- Icons are saved with screen-relative metadata so DeskRealm can best-effort adapt positions when a topology has no exact saved layout yet.
- DeskRealm now guards saves immediately after a display topology change to avoid saving temporary Windows-rearranged positions.
- After the topology settles, DeskRealm restores the current realm layout before accepting future saves.

## Notes

Existing v0.5.0-v0.5.2 layout files are migrated lazily when saved again. For best results after installing, use **Save icon layout now** once per important realm under your normal monitor/resolution/scale setup.
