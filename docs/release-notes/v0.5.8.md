# DeskRealm v0.5.8

## Safe first-run Desktop association

v0.5.8 fixes the first-run onboarding model introduced in v0.5.7. DeskRealm no longer moves files from the original Windows Desktop into a managed realm folder. Instead, the selected realm points directly to the original Desktop folder.

## What changed

- First-run wizard now links the original Windows Desktop path to a selected virtual desktop realm.
- Existing Desktop files and shortcuts remain exactly where they are.
- Closing or disabling DeskRealm restores the original Desktop path, so users keep their normal Desktop.
- Realm assignments now support absolute external paths for safe original-Desktop linking.
- Config migration v6 disables the legacy file-move import flag.
- Duplicate original-Desktop assignments are rejected explicitly.

## Why this matters

The original Desktop is user-owned data. DeskRealm should redirect the known Desktop folder, not migrate user files implicitly during onboarding.

## Upgrade notes

Existing users are not interrupted by the first-run wizard. If you tested v0.5.7 and moved Desktop files into a realm manually, move them back to your original Desktop folder before relying on the v0.5.8 association model.
