# DeskRealm v0.5.4

DeskRealm v0.5.4 focuses on icon layout stability during fast virtual desktop switches.

## Fixed

- Icon layouts are no longer restored immediately after changing the known Desktop folder. DeskRealm now waits for Explorer to settle on the target realm before applying the saved layout.
- Icon layout saves are blocked while a switch restore is pending, avoiding captures where the current Windows virtual desktop and the still-visible Desktop icons are temporarily out of phase.
- Matching icons are positioned in one batched Shell call instead of one by one, reducing mid-restore reflow and cross-icon drift.

## Changed

- Added config v4 fields:
  - `iconLayoutSwitchRestoreDelayMs`
  - `iconLayoutRestoreRetryCount`
  - `iconLayoutRestoreRetryDelayMs`

## Test focus

- Same shortcut present on multiple realms at different positions.
- Fast jumps such as desktop 3 → desktop 1.
- Multi-monitor / resolution changes combined with desktop switches.
