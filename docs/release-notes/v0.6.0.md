# DeskRealm v0.6.0 — Adaptive performance pipeline

`v0.6.0` is the current public release.

> **Historical release note:** `v0.6.0` remains the published stable baseline. The separate Realm Studio successor is tracked as the unreleased [`v0.7.0.md`](v0.7.0.md) development milestone and is not part of this release note.

## User-visible performance gains

- Direct DeskRealm hotkeys no longer wait for Windows navigation to finish before preparing the target realm. The target folder, topology variant and layout restore can begin while Windows is still animating the desktop switch.
- Startup layout recovery now restores the active realm once even when Windows already left the Desktop Known Folder on that realm after a forced close or reboot.
- The UI is less likely to freeze because Shell/state mutations are serialized through a background lane instead of running from the visible form.
- Icon restore is less fragile because DeskRealm waits for the actual target realm contents instead of accepting path-retargeted Explorer views that may still contain old icons.
- Multi-monitor layouts are safer: manual save now replaces only the currently active display-topology variant and integrity-checks all other variants before writing.

## What changed

- Virtual desktop changes are driven by registry notifications instead of a periodic UI timer.
- Switch/save/restore and matching main-window lock/import actions run in one serialized background lane.
- Direct desktop hotkeys wait for released physical modifiers, then run confirmed Windows navigation and destination realm loading/restoration concurrently. The final active GUID validates the transaction; a mismatch triggers explicit compensation to the actual desktop realm.
- One persistent STA worker handles icon-layout operations through a strict BOM-free JSON-lines protocol.
- Explorer must expose the exact target realm membership before placement starts. The worker enumerates view PIDLs through `IEnumIDList` and tolerates bounded `E_BOUNDS` / `E_CHANGED_STATE` transitions while Explorer replaces the collection. During automatic multi-desktop jumps, transition-scoped `GetItemPosition` `E_FAIL` is retried within the same readiness deadline instead of disabling persistence.
- Saved icon positions are applied and verified adaptively.
- The first reconciliation of each process restores the active realm once even when the Windows Desktop Known Folder already points to that realm, repairing reboot/logon layouts after an unclean previous shutdown.
- Shell refresh is targeted to the realm directory; no global broadcast is used.
- Runtime logs include `[PERF]` timings and coalesced notification counts.
- Manual **Save icon layout now** uses a dedicated `save-current-variant` worker command. Only the exact active topology variant can be replaced; all other variants are fingerprinted before/after serialization and must remain unchanged.
- Creating a new topology no longer silently evicts the oldest saved variant when the 24-variant limit is reached; explicit deletion is required.
- Source builds target .NET 10 / C# 14; release artifacts remain self-contained `win-x64`.

## Preserved UX foundations

- Embedded DeskRealm icon and README demo.
- First-run onboarding and safe original Desktop association.
- Capture-based hotkey editor.
- Layout, realm and topology-variant locks.
- Multi-display topology details and variant deletion.

## Validation status

The source package passes the repository's static consistency checks. Local runtime testing validated startup recovery, locked-variant save integrity and the parallel hotkey behavior before publication. A final Windows Release build/check should still be run before attaching published binaries to the GitHub release.
