# DeskRealm v0.5.6

Current stable hotfix for icon layout restoration.

## Fixed

- Stabilized icon layout restoration when the same shortcut/icon exists on several realms with different positions.
- Added Shell identity fallback: exact PIDL matching is still preferred, but DeskRealm can now match by Shell display/parsing identity when Explorer exposes the same shortcut with a different PIDL.
- Improved diagnostics by storing/logging human-readable Shell identity metadata instead of only `Desktop item #...` placeholders.
- Keeps the v0.5.3-v0.5.5 protections for display topology, DPI / scale, deferred restore, verified restore and retry.

## Recommended after upgrade

Use **Save icon layout now** once on each important realm so layouts are refreshed with the v0.5.6 identity metadata.

## Validation focus

- Same icon on multiple realms with different positions.
- Multi-monitor setup with one monitor temporarily off.
- Resolution changes.
- Windows display scale / DPI changes.
- Fast virtual desktop jumps.
