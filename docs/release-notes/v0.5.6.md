# DeskRealm v0.5.6

DeskRealm v0.5.6 improves icon identity matching when the same shortcut/icon appears on multiple realms or when Explorer exposes an item with a different PIDL after a desktop/display transition.

## Fixed

- Added secondary Shell identity metadata for captured desktop icons.
- Restores by exact PIDL first, then falls back to Shell display/parsing identity when the exact PIDL is not visible.
- Logs fallback matches with human-readable names so persistent icon issues can be diagnosed without guessing which `Desktop item #...` was involved.

## Notes

After installing v0.5.6, run **Save icon layout now** once on each realm that has repeated shortcuts. Older layout files do not contain the new identity metadata, so one fresh save per realm is required for the fallback matcher to become useful.
