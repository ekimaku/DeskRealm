# DeskRealm v0.7.0 — release-control validation

Date: 2026-06-25  
Application version: `0.7.0`

## Scope

This document freezes the public release contract for v0.7.0: **Realm Studio — Your Windows Workspaces, Made Visible**. It consolidates the Realm Studio modernization, Explorer restart/tray recovery, direct realm-card controls, config migration and release-pipeline simplification into one public release.

## User-validated Windows behavior

The v0.7.0 candidate chain was used in normal day-to-day Windows use for multiple days. The following behavior was reported working in the final local candidate:

- self-contained `Build-Release.ps1` output builds and launches;
- Realm Studio loads and updates realm cards correctly;
- Windows wallpaper changes synchronize into DeskRealm previews, and DeskRealm wallpaper saves apply correctly;
- native/original Desktop realm editing preserves the physical Desktop folder;
- virtual-desktop rename applies after Explorer restart or reboot;
- DeskRealm tray icon returns after Explorer recreates the notification area;
- hotkeys, direct wallpaper controls, inline hotkey capture, lock/default controls and **Start minimized** work;
- the false transient `Desktop 1` / snapshot conflict after Explorer restart is resolved.

## Release package checks

- Version alignment: `VERSION.txt` and `DeskRealm.App.csproj` declare `0.7.0`; `CHANGELOG.md` and `docs/release-notes/v0.7.0.md` use the titled public heading `v0.7.0`.
- PowerShell compatibility: release-helper script sources are ASCII-only; the required CHANGELOG em dash is generated at runtime rather than stored as a raw script character.
- README media alignment: the first GIF demonstrates realm switching/layout continuity and the GIF directly below it demonstrates Realm Studio direct controls.
- Distribution remains unpackaged, self-contained `win-x64`.
- GitHub Actions builds Windows artifacts for `main`, pull requests and tags. Only the tag-only release job receives `contents: write`.
- Release assets include portable and install-bundle ZIPs plus `SHA256SUMS.txt`.
- No `bin`, `obj`, `dist`, `.git`, build artifacts or nested source archive is included in the source ZIP.

## Final publication gate

Run the following from the real Git worktree before tagging:

```powershell
.\scripts\Prepare-GitHubRelease.ps1 -Version 0.7.0
```

Then verify the portable EXE once more, review `git status`, commit the release-ready tree, create an annotated `v0.7.0` tag and push `main` plus the tag. GitHub Actions is the final independent build/release gate.

## Deferred, not blocking v0.7.0

- Dedicated automated test project and focused migration fixtures.
- Service extraction from the large switching/Shell services.
- Sanitized diagnostics support bundle.
- Installer/upgrade compatibility matrix and code-signing evaluation.

Those items are planned in `docs/ROADMAP.md` and are intentionally not mixed into the release freeze.
