# DeskRealm v0.7.0 Inline Hotkey Layout & First-Click Repair — Static Validation (`_bh`)

## Scope

- VCard inline hotkey first-click render ordering.
- Full-width hotkey capture field and separate action row.
- Reset / Save / Cancel draft boundaries.

## Performed

- Parsed all `.xaml`, `.csproj` and `.props` XML files.
- Parsed JSON files.
- Ran a C# delimiter scan across source files.
- Verified the inline capture GUID is set before `SuspendGlobalHotkeysForCaptureAsync`.
- Verified the VCard binds `ResetHotkeyCommand`, `HotkeyEditorActionsVisibility` and no longer renders a VCard `Clear` button.
- Verified the reusable field exposes `ResetToInitialValue()` and explicit `Waiting input...` state.
- Verified a single `DeskRealm/` archive root and excluded generated output.

## Not performed in this environment

- .NET / WinUI 3 compilation.
- Windows App SDK PRI generation.
- Windows Shell, tray, hotkey or Explorer runtime smoke tests.

## Required Windows validation

Run `./scripts/Build-Release.ps1`, then the `_bh` section of `SMOKE_TEST.md`.
